# DESIGN_GUI_EXECUTION_PROTOCOL

> Status: active
> Authority: authoritative
> Read-Tier: task-scoped
> Purpose: 定义 GUI 任务分解、切片交付、验收门禁与前后端字段归属决策流程。

## 1. 任务分解规则
- 页面清单用于开发分工。
- 任务流用于功能验收。
- 合并前必须确认“任务流闭环已成立”，不能仅以页面完成度作为通过条件。

## 2. 垂直切片交付规则
- 每次交付一条最小可用路径，覆盖 UI、状态、API、持久化、错误处理。
- 切片验收条件：
  1. 主路径 100% 跑通。
  2. 受影响相邻路径完成回归。
  3. 关键错误可见且可恢复。
  4. 至少有最小自动化测试覆盖。

## 3. 前后端字段归属决策卡（必填）
- 字段名
- 业务用途
- 生命周期（临时/会话/长期）
- 权威端（前端/后端）
- 同步策略（强一致/最终一致/混合）
- 回滚策略
- 兼容迁移策略

未完成决策卡评审的字段，禁止进入实现代码。

## 4. 评审与验收节奏
- 每日短评审必须查看可运行结果。
- 验收必须覆盖：加载态、空态、错误态、刷新一致性。
- 文档用于冻结规则，不替代运行态验收。

## 5. 提交与回滚
- 分阶段提交，禁止跨阶段混改。
- 每阶段提交后应保持可运行、可测试。
- 回滚单位按阶段切分，保证可快速恢复。
