# repo_memo/archive

> Status: stable
> Authority: authoritative
> Read-Tier: on-demand
> Purpose: 定义历史归档目录的用途、读取规则与与主目录/未来方案目录的边界

本目录存放已完成且已被其他权威文档吸收的历史设计、阶段决策与同步记录。

## 用途
- 保留历史背景、阶段决策与迁移痕迹，供追溯与审计使用
- 为后续复盘、冲突排查、历史比对提供来源
- 不作为当前实现的默认输入

## 读取规则
- agent 默认不主动读取本目录
- 仅在以下场景按需读取：
  - 需要追溯某项历史决策的来源
  - 需要比对某项规范的演进过程
  - 需要核查归档前后是否遗漏有效约束

## 与其他目录的区别

| 目录 | 说明 |
|------|------|
| `repo_memo/` 主目录 | 当前有效的权威规范 |
| `archive/` | 历史归档，默认不作为实现依据 |
| `further/` | 已确认但当前不执行的未来方案 |
