## 设计决策（Phase 0）

### 核心原则确立
- **后端是唯一权威数据源**。前端不做业务数据持久化。Pinia store 仅暂存内存状态。
- **REST API 是权威接口**。mock 也在 API 层拦截（MSW），不另设 Repository 抽象。
- **页面解耦**：选项卡间禁止直接调用，通过 REST API / persistence（仅 UI 状态）通信。

### 数据模型重定义

三文件职责矩阵：
| 文件 | 职责 |
|------|------|
| database.json | 纯磁盘扫描数据（steamlib/game/mod），无 managed/warnings/errors |
| workspace.json | branch_decisions + 计算摘要（inputs_hash）+ 扫描参数 |
| user_config.json | bakprefix/bakignore/output_paths/rule_sources |

**managed_entries**：不在 database，不在 workspace。作为 `POST /api/pipeline/compute` 的可选参数（列表格式——"仅保留这些路径"）。

### 六页面流

```
DataSource → RulesOverview → 计算准备 → Forest → Conflicts → Operations
                + Settings       + 高级（进阶用户）
```

- DataSource：纯展示，无裁决
- RulesOverview：选规则 + 聚合（kmmrule 文件结构修正：nickname/preview/readme 在 mod 级别）
- 计算准备：可选预选（checkbox）→ 构建 managed_entries → 触发 compute
- Forest：全屏 SVG + 顶栏 3 按钮 + 3 浮层（小地图/底栏/抽屉），overflow:hidden
- Conflicts：branch_decisions 裁决
- Operations：消费结果 → 备份/应用/恢复

### Forest 全屏布局
- 顶栏：🔄 重置视图 | 📐 小地图 | 📊 摘要 (n⚠)
- 底栏按钮 📋 n（左下角 toggle）- 留存 UI 状态
