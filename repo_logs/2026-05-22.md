### 3.2 Bootstrap 实现改造 (`f27627c`、`a9c1e85`)

**`src/modmgr/bootstrap.py`**：
- 新增 macOS 配置目录：`~/Library/Preferences/kmm/user_config.json`
- Windows database 路径修正：`%LOCALAPPDATA%/kmm/database/database.json`
- macOS database 默认路径：`~/Library/Application Support/kmm/database.json`

**`src/modmgr/orchestrator/_common.py`**：
- `_default_workspace_dir()` 新增三平台逻辑：
  - Linux: `~/.cache/kmm/workspace/`
  - Windows: `%LOCALAPPDATA%/kmm/workspace/`
  - macOS: `~/Library/Caches/kmm/workspace/`

**`src/modmgr/orchestrator/resolver.py`**：
- `_default_ws_dir()` 同步三平台逻辑

### 3.3 三平台默认路径汇总

| 用途 | Linux | Windows | macOS |
|------|-------|---------|-------|
| `user_config` | `~/.config/kmm/` | `%APPDATA%/kmm/` | `~/Library/Preferences/kmm/` |
| `database` | `~/.local/share/kmm/` | `%LOCALAPPDATA%/kmm/database/` | `~/Library/Application Support/kmm/` |
| `workspace` | `~/.cache/kmm/workspace/` | `%LOCALAPPDATA%/kmm/workspace/` | `~/Library/Caches/kmm/workspace/` |

### 3.4 手动路径指定

`generate_database(mode="manual", paths=[...])` 已存在，接受 steamapps 目录或 VDF 文件路径列表。此机制在 DESIGN_BOOTSTRAP.md §二 中明确记录。Steam.exe → vdf 路径推算在此阶段作为文档建议保留（`steam.exe` 所在目录下 `config/loginusers.vdf` → 推算 `libraryfolders.vdf`）。

### 3.5 废弃字段清理 (`d2ea6ee`)

`aggregated_ruleset_output_path` 从 schema / 代码 / 前端 / 文档中全面移除。聚合规则不再写入独立文件，改为工作区目录下的 `aggregated_rule.json`。
